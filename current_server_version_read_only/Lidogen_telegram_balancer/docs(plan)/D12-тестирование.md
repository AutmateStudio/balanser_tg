# D12 — тестирование (E2E staging: API → PG → worker → clump)

> Критерий приёмки MVP (спринт S3).  
> Скрипты и чеклист: `scripts/e2e_d12/` (`preflight_d12.py`, `run_e2e_d12.py`, `checklist.md`).  
> Доступ к БД: [`db-access-via-tailscale.md`](db-access-via-tailscale.md).

## Задача D12

Сквозной сценарий на staging:

1. `POST /discovery-api/parser/{id}/add-channels?async=true` с **D8** (`USE_PG_QUEUE=true`) → задачи в PostgreSQL.
2. **queue-worker** на vps-101 выполняет `parser_add_channel` через Telethon clump.
3. Статус задачи **`done`**, канал в clump, опционально сверка PG (D5 usage, B9 schema).

Зависимости: **D8**, **D3**, **D5**, **D10**, **C2**; **B9** — schema + repo (история `task_attempts` в E2E опциональна).

---

## Три компонента (все должны работать)

| # | Компонент | Где обычно | Что делает |
|---|-----------|------------|------------|
| 1 | **standalone_discovery** (FastAPI + Telethon + VPN) | Staging (напр. `194.156.117.160:8100`) | HTTP API, clump в памяти, **D8:** enqueue в PG |
| 2 | **queue-worker** (balancer) | **vps-101** | Claim из PG, **clump** adapter |
| 3 | **PostgreSQL** `lead_monitor` | **vps-100** (Tailscale) | `task_queue`, `accounts`, … |

```text
  run_e2e_d12.py / curl
        │
        ▼
  Discovery API  ──D8 enqueue──►  PostgreSQL (vps-100)
        │                              │
        │ clump в памяти              │ claim / complete
        │                              ▼
        │                         queue-worker (vps-101)
        │                              │
        └──────── Telethon add ────────┘ (ClumpTaskAdapter)
```

**На vps-101 не поднимается HTTP discovery** — только воркер и pytest.  
**standalone_discovery** запускается на **staging-сервере** (отдельный `docker compose` в `standalone_discovery/`).

## Этап 2. standalone_discovery на staging (обязательно)

### 2.1. Сборка image с PG-очередью (D8)

Стандартный `standalone_discovery/Dockerfile` **не** включает `app_balance`.

```bash
cd ~/Lidogen_telegram_balancer
docker build -f standalone_discovery/Dockerfile.pg-queue -t standalone-discovery-api:latest .

cd standalone_discovery
cp .env.example .env
nano .env
```

Минимум в **`standalone_discovery/.env`**:

```env
API_ID=...
API_HASH=...
API_KEY=...

USE_PG_QUEUE=true
QUEUE_DATABASE_URL=postgresql://lead_monitor_owner:ПАРОЛЬ@vps-100:5432/lead_monitor
DISCOVERY_APP_PORT=8100
```

На сервере (не в git): `vpn_telegram_gleb.conf`, `sessions/*.session`, при необходимости `data/parser_jobs.json`.

### 2.2. Запуск VPN + discovery-api

```bash
cd ~/Lidogen_telegram_balancer/standalone_discovery
docker compose up -d
docker compose ps
docker compose logs -f discovery-api
```

Проверки:

```bash
curl -sS http://127.0.0.1:8100/health

curl -sS -H "X-API-Key: ВАШ_API_KEY" \
  http://194.156.117.160:8100/discovery-api/parser/list
```

### 2.3. Запуск clump (`POST /parser/start`)

Без запущенного парсера `add-channels` не работает (404/409).

```bash
curl -sS -X POST \
  -H "X-API-Key: ВАШ_API_KEY" \
  -H "Content-Type: application/json" \
  http://194.156.117.160:8100/discovery-api/parser/start \
  -d '{
    "session_name": "/app/sessions/Client1",
    "channel_list": ["@уже_известный_канал"],
    "webhook_url": "https://your-n8n.example.com/webhook/telegram"
  }'
```

Сохранить **`parser_id`** → `scripts/e2e_d12/env.d12` → `PARSER_ID`.

```bash
curl -sS -H "X-API-Key: ВАШ_API_KEY" \
  http://194.156.117.160:8100/discovery-api/parser/status/PARSER_ID
```

### Флаги E2E (`env.d12`)

| Переменная | По умолчанию | Назначение |
|------------|--------------|------------|
| `E2E_VERIFY_TASK_ATTEMPTS` | `false` | B9: ждать строки в `task_attempts` |
| `E2E_SKIP_VERIFY_PG` | `false` | Пропустить сверку PG |
| `E2E_POLL_TIMEOUT_SECONDS` | `180` | Таймаут до `status=done` |

## Порядок запуска (кратко)

1. PG: `migrate` + `sync_accounts_to_pg` на vps-101

На discovery временно `USE_PG_QUEUE=false`, перезапуск контейнера:

- `async_mode=true`, `action_id` из SQLite
- **нет** `task_ids` (или пустой список)

## Типичные ошибки

| Симптом | Что проверить |
|---------|----------------|
| `undefined volume` при `docker compose build` | `HOST_SESSIONS_DIR=./standalone_discovery/sessions` |
| Discovery недоступен | `standalone_discovery` не запущен, порт, firewall |
| 401 | `DISCOVERY_API_KEY` ≠ `API_KEY` discovery |
| Нет `task_ids` | `USE_PG_QUEUE=true`, image `Dockerfile.pg-queue` |
| 500 на add-channels | Image без `app_balance` |
| Задача `queued` долго | Воркер не запущен или `WORKER_TASK_ADAPTER=mock` |
| `failed` | Логи worker + discovery; канал / flood |
| Пустые `task_ids` | dedup — другой канал или дождаться `done` |
