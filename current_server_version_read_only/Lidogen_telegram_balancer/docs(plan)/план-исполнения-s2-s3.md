## 1. Цель итерации

Собрать **полный цикл dispatch воркера** (claim → reserve → проверка ресурса → execute → release/postpone/retry)
и довести до **MVP: add-channels через PostgreSQL-очередь** с реальным Telethon-адаптером.

Definition of Done итерации:

- воркер не no-op: резервирует аккаунт, вызывает адаптер (сначала mock, затем Telethon);
- при отсутствии аккаунта / нехватке ресурса задача **postpone**, очередь не блокируется;
- `move_channel` резервирует source + target атомарно;
- watchdog помечает зависшие задачи `stuck`;
- интеграционные тесты C8/C9 зелёные;
- `USE_PG_QUEUE=true`: `POST /add-channels` → N задач в PG → worker → clump;
- flood/ban из HealthMonitor попадают в PG (`cooldown` / `banned`);
- E2E на staging: один реальный канал добавлен через PG-очередь.
