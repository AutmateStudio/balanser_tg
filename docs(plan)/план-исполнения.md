## 1. Цель итерации

Поднять слой доступа к PostgreSQL-очереди (пул + репозитории), точечные методы исполнения в `SessionClump`
и мост аккаунтов discovery → PG, чтобы следующим шагом (C2–C9, D3+) собрать worker с mock-адаптером.

Definition of Done итерации (по §5 итогового плана, частично S1 + начало S2):

- миграции и seed применяются одной командой (уже готово — A11);
- `queue/db.py` даёт пул и healthcheck;
- репозитории `task_types`, `task_queue` (enqueue/dedup/claim), `accounts` (pick/reserve/release), `resource_usage` (per-op) покрыты тестами;
- `SessionClump` умеет добавить канал на конкретную сессию и перенести один канал;
- скрипт sync аккаунтов идемпотентен;
- `docker compose up queue-worker` стартует процесс воркера (минимальный loop из C1).

### B8 — `repos/resource_usage.py`: insert, count_last_hour (per-op) ✅ ГОТОВА

- **Приёмка:** `count_last_hour` по `op_type_id` совпадает с суммой `units` за час; «худший op» возвращает минимальный `available_percent`.
